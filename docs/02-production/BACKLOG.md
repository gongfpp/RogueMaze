## WP-001 详细定义

目标：在不引入引擎和 UI 的前提下，证明规则模型可测试、可配置，并提前发现“路如何连接”和“手牌如何循环”的歧义。

范围：

- 四方向端口和 5 种基础路块的数据表示。
- 放置边界、重叠与相邻端口匹配规则。
- 固定顺序循环、随机弃牌洗回两种牌组策略。
- 给定起点/终点/牌序的最小回合模拟。
- 自动测试和简短实验报告。

不含：Godot 节点、渲染、触控、声音、完整关卡生成。

验收标准：

1. 合法拼接、断口、越界和重叠分别有测试。
2. 两种循环模式在固定随机种子下可重放。
3. 至少三个关卡样例能得出胜/负结果和明确原因。
4. 核心规则不依赖 UI 或引擎类型。
5. 更新开发日志、架构文档、变更记录和对应实战教学。

完成证据：

- `npm test`：15/15 通过。
- `npm run simulate:rules`：平路胜利、爬坡胜利、接口不匹配失败，结果与样例预期一致。
- 固定循环保持作者牌序；弃牌洗回在相同种子下得到相同序列。
