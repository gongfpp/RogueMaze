# ADR-0001：引擎与基础技术栈

- 状态：已接受
- 日期：2026-07-12
- 决策者：项目发起人与开发协作流程

## 背景

项目需要制作 2D、移动端触屏优先，同时支持 Android、iOS、Windows、Linux 的短局游戏。当前仓库为空，核心风险在玩法验证和长期可维护性，而不是高端 3D 渲染。

## 决策

采用 Godot 4.7 stable 标准版、GDScript 和 Compatibility 渲染。规则核心保持不依赖场景树的纯逻辑，便于测试和四平台复用。

## 理由

- 2D 工作流、场景组合和快速迭代适合本项目。
- 开源且无运行时抽成，早期试错成本低。
- GDScript 上手和迭代速度快，足以支持当前复杂度。
- Android、iOS、Windows、Linux 都有正式导出路径；但 iOS 仍依赖 Apple 工具链。

## 代价

- 商业 SDK 和第三方移动插件生态可能不如 Unity 丰富。
- 四个平台的导出、输入差异、插件兼容性与目标设备性能必须尽早实测。
- 纯逻辑层与 Godot 表现层的边界需要主动维护。

## 备选方案

- Unity + C#：若团队已有成熟 Unity 经验、需要大量商业 SDK 或主机扩展，可重新评估。
- Flutter/游戏框架：应用 UI 强，但游戏场景、资源与编辑器工作流不占优。
- 各平台原生开发：控制力高，但四套工程的开发和维护成本不适合当前阶段。

## 验证条件

`WP-002` 已确认：Godot 使用 MIT 许可证；Windows/Linux 可由当前工具链导出；Android 还缺 SDK/ADB 与真机；iOS 必须使用 macOS、Xcode、签名账号与真机。当前没有否决本方案的阻塞。
