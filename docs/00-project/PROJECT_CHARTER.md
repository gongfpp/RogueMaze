# 项目章程

| 项目项 | 内容 |
| --- | --- |
| 项目代号 | RogueMaze |
| 产品类型 | 2D 实时拼路 + 卡牌构筑 + Roguelite 过关 |
| 目标平台 | Android、iOS、Windows、Linux |
| 当前阶段 | P0 立项与预制作 |
| 策划基线 | v0.1，2026-07-12 |
| 首发模式 | 单机、短局制；移动端竖屏，桌面端自适应窗口 |
| 暂定引擎 | Godot 4.x + GDScript |

## 1. 产品愿景

玩家不是直接控制奔跑者，而是在时间压力下为他“创造脚下的世界”。每一次抽牌都既是路线选择，也是风险管理。游戏应让玩家在几秒内读懂局面、做出取舍，并在失败后立刻想再来一局。

一句话体验目标：**用有限而循环的道路卡牌，在不断前进的压力下拼出一条聪明、惊险且独一无二的生路。**

## 2. 目标玩家

- 喜欢轻策略、卡牌构筑、拼图与 Roguelite 的移动端玩家。
- 可接受 3–8 分钟一局，希望单手操作、随时暂停或重开。
- 不要求动作游戏反应速度，但愿意观察、预判和构筑组合。

## 3. 设计支柱

1. **看得懂**：道路连接、危险范围和落点必须一眼可读。
2. **来不及但来得及**：持续前进制造张力，慢速、暂停型能力提供思考窗口。
3. **每张路都有取舍**：道路不只是形状，还可能带有速度、资源、机关和耐久属性。
4. **失败可复盘**：失败原因明确，玩家知道是路线、时机还是构筑问题。
5. **小规则产生大变化**：用单位路块、词缀和事件组合产生关卡变化，避免依赖海量手工内容。

## 4. 成功标准

### 原型阶段

- 新玩家无需文字教学，30 秒内能完成“选牌—放路—角色通行”。
- 5 分钟测试中至少出现 3 次有意义的路线选择。
- 失败后，测试者能准确说出主要失败原因。

### MVP 阶段

- 从启动到开始首局不超过 3 次点击。
- 中端目标设备稳定达到 60 FPS；低端降级目标 30 FPS。
- 完整流程包含：新局、3 个节点、1 次卡牌选择、Boss/终点、结算与重开。
- Android、iOS 真机以及 Windows、Linux 桌面构建均完成冒烟测试，且存档可正确恢复。

### 软发布阶段

留存、关卡完成率、崩溃率等阈值在埋点原型获得真实数据后制定，避免当前阶段凭空承诺商业指标。

## 5. 项目约束与假设

- 以小团队/个人开发为基线，AI 辅助策划、编码、测试与文档。
- 移动端采用竖屏单手操作；Windows/Linux 支持鼠标、键盘和自适应窗口，桌面布局不简单拉伸手机画面。
- 首发单机，不把联网、PvP、公会、UGC 纳入 MVP。
- 首版使用简洁几何美术，优先验证玩法而非资产量。
- iOS 构建最终需要 macOS、Xcode、开发者账号与真机；在对应里程碑前确认资源。
- Linux 至少验证一个主流 64 位发行版；Windows 至少验证 Windows 10/11 64 位。具体最低版本在 Beta 前根据引擎支持范围锁定。
- 排期是决策基线而非承诺日期，每个阶段通过门禁后再细化下一阶段。

## 6. 决策原则

优先级依次为：核心循环可玩性 > 操作与可读性 > 稳定性 > 内容丰富度 > 商业化。新增想法先进入候选池，除非解决当前里程碑的明确问题，否则不立即开发。

## 7. 阶段门禁

| 门禁 | 通过条件 |
| --- | --- |
| G0 立项完成 | 章程、GDD、MVP、路线图、风险与流程已评审 |
| G1 核心可玩 | 自动行进、放路、连接判定、胜负闭环可试玩 |
| G2 垂直切片 | 一局完整、基本音画、至少 1 个构筑选择与 1 个特色机关 |
| G3 Alpha | MVP 功能齐全，内容可从数据配置，主流程无阻断 |
| G4 Beta | 四个平台构建通过，性能、存档、输入适配与新手引导达标 |

## 8. 长期协作约定

- 每轮开发必须至少产生一次有说明的 Git commit，提交前先验证本轮交付。
- 远端统一使用 `https://github.com/gongfpp/RogueMaze.git`，不另建互相分叉的“正式版本”。
- 一轮中可以连续完成多个小工作包，但每个包都要有验收结果，不能把大量半成品混成一次提交。
- 能安全自动完成的检查、实现和文档更新直接进行；只有权限、破坏性操作、付费或外部账号等真实阻塞才请求确认。
- 文档先帮助读者做成事情，再解释术语；一句普通话能讲清楚的内容，不换成一串行话。
- 教程默认读者可能从未写过游戏，必须解释基础概念、给出运行方法，并说明模块怎样交接给下一位成员。
| G5 发布候选 | 商店素材、合规、崩溃监控、回滚与发布清单完成 |
