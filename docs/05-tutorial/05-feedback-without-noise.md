# 第 05 章：背景、动画和音效怎样不破坏玩法

这一章写给第一次参与游戏表现制作的成员。重点不是“做得越多越好”，而是每个视觉和声音变化都帮助玩家理解发生了什么。

## 1. 先列信息优先级

RogueMaze 的画面优先级是：

1. 角色下一步有没有路。
2. 当前牌能放在哪。
3. 危险何时生效。
4. 手牌内容和选中状态。
5. 世界气氛和装饰。

背景纹理属于第 5 层。如果背景黄铜线比道路还亮，即使图片很漂亮，也是失败的游戏美术。

## 2. 怎样接入一张背景

生成图是正方形，游戏窗口是竖屏。直接拉伸会让纸张纹理变形，所以代码先计算目标宽高比，从正方形中央裁出对应区域，再铺满屏幕。

之后叠加深蓝色蒙层。当前透明度是 78%，来自 405×720 截图对比，而不是“美术规范一定要 78%”。以后背景变化需要重新检查。

## 3. 资产来源为什么要和图片放在一起

`assets/art/README.md` 记录文件用途、来源、哈希、生成提示摘要和限制。这样半年后接手的人能回答：

- 这张图能否用于发布？
- 为什么必须加深色蒙层？
- 新版本应该覆盖还是另存？
- 文件有没有被意外替换？

“我记得好像能用”不是可交付的授权记录。

## 4. 动画只表达状态变化

- 选牌抬起：告诉玩家“操作对象是它”。
- 放路脉冲：告诉玩家“放置已被接受”。
- 角色步态：告诉玩家“时间正在推进”。
- 岩石呼吸：告诉玩家“危险尚未触发但在逼近”。

这些动画不改变网格坐标。规则层仍只看 Vector2i，表现层只在画图时增加几像素偏移。

## 5. 没有音频文件也能验证声音时机

程序可以生成一小段波形：

```text
正弦波 × 音量衰减包络 → 16-bit PCM 字节 → AudioStreamWAV
```

正弦波提供音高，衰减包络让声音快速消失。再加一点二倍频，听起来更像机械轻响。放路用较高短音，落石用较低长音。

它的优势是没有素材授权风险、体积极小、参数可测试。缺点是音色简单，所以发布前仍需要声音设计与真机扬声器检查。

## 6. 规则怎样通知声音

GameSession 不直接播放声音，而是记录事件：

```text
SPIKE_HIT → 表现层 → 受伤音 + 后续震动
ROCK_FELL → 表现层 → 低频落石音 + 屏幕反馈
```

测试可以检查事件存在；静音设置可以让表现层不播放；规则结果完全相同。

## 7. 音效开关和设置的区别

当前 `SFX ON/OFF` 只改变本次运行中的 `AudioCues.enabled`。关闭再启动会恢复默认，这叫临时状态。

真正的设置需要写入 `user://`，带数据版本并在启动时读取。不能因为按钮看起来能用，就声称“设置系统完成”。

## 8. 交付一个表现资产

提交背景、音效或动画时至少附带：

- 它反馈哪个游戏事件。
- 来源和授权记录。
- 移动窄屏与桌面截图。
- 对比度、音量或性能风险。
- 关闭该表现后规则是否仍正确。

表现模块的价值是让规则更容易理解，不是让工程更难接手。
