# 第 11 章：玩家切到微信时，游戏不能偷偷继续

## 1. 生命周期是什么

应用不只有“开着”和“关了”。手机可能把它放到后台、暂停渲染、恢复，甚至在内存紧张时直接结束。桌面窗口也会失去焦点。

对 RogueMaze 来说，最安全的默认行为是：离开就暂停并保存。因为角色会自动前进，继续计算会把一次正常切屏变成失败惩罚。

## 2. 幂等为什么重要

“设置为暂停”与“切换暂停”不同。系统可能连续发两次离开通知；如果每次都执行切换，第一次暂停，第二次反而恢复。

```text
set_paused(true)   → 无论调用几次，结果都是暂停
toggle_pause()     → 每调用一次就反转，只适合玩家按钮
```

系统事件使用前者，玩家按钮可以使用后者。

## 3. 保存两个文件时不要短路

如果写成 `save_settings() and save_progress()`，某些语言会在设置保存失败后不再执行进度保存。项目先分别执行两次，再合并结果。这样磁盘偶发问题不会让本来还能保存的另一份数据也被跳过。

## 4. CI 是另一台干净电脑

持续集成（CI）可以理解为：每次提交后，让一台没有你本地缓存和手工配置的新电脑照文档执行。它能发现“我电脑上刚好装过这个”“文件大小写在 Windows 没事、Linux 找不到”等问题。

本项目 PR 先跑 Node 与 Godot 测试；主分支再导出 Linux 成品。导出模板超过 1 GB，所以缓存锁定的 4.7 模板，既节省时间又不偷偷升级。

## 5. 工作流文件存在，不代表 CI 通过

`.github/workflows/ci.yml` 只是自动化说明。必须推送到 GitHub，看到具体运行记录和绿色任务，才能写“CI 通过”。本地只能检查 YAML 内容和复现其中命令。

这条区分和真机测试一样重要：配置证据、执行证据、产物证据不能混在一句“支持”里。
