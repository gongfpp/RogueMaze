# 游戏开发实战教学

这套教学写给游戏开发实习生，也写给暂时不会编程、但想参与项目的人。它跟随 RogueMaze 的真实开发逐章增长，不假装所有答案一开始就知道。每章会解释基础概念、带读者做出可运行结果，并记录真实的取舍和踩坑。

## 完全没有开发经验

先读 [零基础成员从这里开始](START_HERE.md)。读完不要求你能独立写游戏，但你应该能看懂项目由哪些部分组成、Git 在保护什么，以及怎样领取一个不会失控的小模块。

## 学习路线

1. [第 00 章：把游戏点子变成可执行项目](00-from-idea-to-project.md)
2. 第 01 章：用纯逻辑验证拼路规则（随 `WP-001` 产出）
3. 第 02 章：搭建可测试的 Godot 2D 工程（随 `WP-010` 产出）
4. 第 03 章：网格、端口与数据驱动路块（随 `WP-011` 产出）
5. 第 04 章：让角色稳定沿路径自动前进（随 `WP-012` 产出）
6. 第 05 章：移动端拖拽、吸附与输入反馈（随 `WP-013` 产出）
7. 第 06 章：构建可复现的卡牌循环（随 `WP-014` 产出）
8. 后续章节按里程碑继续补充，不预写未经实践的结论。

## 每章标准

- 明确本章解决的问题。
- 从验收标准反推设计。
- 展示核心模型和最小实现。
- 包含能复现的测试或试玩方法。
- 区分“项目中的选择”与“普遍适用原则”。
- 记录失败方案和适用边界。
- 第一次出现的基础概念要用普通话和具体例子解释。
- 给出“怎么运行、怎么确认做对了、怎么交给下一位成员”。
