# 第 13 章：免费引擎也要带许可证

## 1. “可以免费商用”不等于“什么都不用做”

Godot 使用 MIT 许可证，允许使用、修改和商业发行，但要求在软件副本或重要部分中保留版权与许可文字。引擎还使用许多第三方组件，因此官方仓库提供完整 `COPYRIGHT.txt`。

这不是在游戏首页塞一整本法律文件。常见做法是：设置或暂停页显示引擎署名，完整文本随安装包或应用资源分发。

## 2. 为什么要锁定版本对应的版权文件

不同 Godot 版本可能更新依赖。RogueMaze 使用 4.7 stable，就从 `4.7-stable` 标签取得清单并记录 SHA-256。升级到 4.7.1 或 4.8 时不能继续沿用旧文件，必须重新下载、复核和构建。

哈希像文件指纹。它不能判断许可证是否适合项目，但能证明团队讨论的是同一份文件。

## 3. 文件放进仓库还不够

Godot 导出通常只收集被识别或引用的资源。一个孤立的 `.txt` 可能留在源码仓库却没进成品。本项目在四个导出预设中明确包含 `assets/legal/*.txt`，并让成品 smoke 用 `FileAccess.file_exists` 再检查一次。

自动测试回答的是“文件真的在包里”，不是“法律判断已经由律师完成”。

## 4. 不要替项目所有者选代码许可证

引擎许可证和游戏代码许可证是两件事。Godot 是 MIT，不代表 RogueMaze 自动也是 MIT。公开仓库没有 LICENSE 时，默认并不等于任何人都能随意复制。

选择 MIT、Apache-2.0、GPL、闭源商业许可或其他方案，会影响贡献和发行权利，必须由拥有权利的人明确决定。实习生可以整理选项，不能自行提交一份 LICENSE 宣布公司资产开源。

## 5. 发布检查

每次更换引擎、字体、音乐、图片、SDK 或商店服务时，都要回答：来源是什么？允许怎样使用？是否要求署名？许可证文本是否进包？隐私权限是否变化？

把答案和资产放在一起，比发行前从聊天记录里考古可靠得多。
