# 第 12 章：绕路不只是躲危险，还要有值得去的东西

## 1. 只有惩罚的选择很容易变成标准答案

如果中路有尖刺，上路什么都没有，玩家只是在判断能不能绕开；一旦学会，上路就成了固定答案。好的路线选择通常同时包含成本和收益：绕路需要特定卡牌、更多铺路时间，但可能获得治疗。

维修台让上坡和下坡不再只是“主路坏了才用”。玩家可能主动规划：先上到维修台，再下回终点；或者牌序太紧，放弃治疗直冲。

## 2. 互动要素需要完整状态

维修台至少有两个状态：可用、已用。规则还要回答：满血会不会消耗？治疗会不会超过最大生命？重复进入会不会无限治疗？

本项目的答案是：只有生命未满才触发；生命最多回到上限；成功治疗后标记 `spent`，不能再用。测试逐项固定这些答案。

## 3. 奖励也需要事件

规则层把生命从 2 改到 3 后发出 `REPAIRED`。UI 收到事件才显示文字和播放正向提示音。不要让 UI 每帧比较“这一帧血量是不是比上一帧高”，那会把读档、调试指令和真正治疗混在一起。

## 4. 用图形和文字重复表达

青色可能与桥梁颜色接近，也可能对部分玩家不明显。因此维修台还有圆环和明显加号，节点顶部提示写着 `TEAL + HEALS`。使用后轮廓变暗。

这叫冗余编码：颜色、形状、文字至少两种同时表达重要信息。危险、强化和治疗都应该这样做。

## 5. 关卡设计练习

在 8×5 棋盘上画一条主路和一条治疗支路。列出支路多用几张牌、玩家何时能看到维修台、最差牌序是否会把角色困在高处、满血玩家还有没有理由走支路。

如果每种状态下答案都一样，这个选择可能只是装饰。先改纸面布局，再改内容 JSON。
