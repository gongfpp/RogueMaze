# 2026-07-12：WP-031 版本化设置与进度存档

## 玩家得到什么

- 关闭 SFX 后，重新启动仍保持关闭。
- `FX FULL/LOW` 控制选牌抬升、角色步态、放路脉冲和落石呼吸，并持久保存。
- HUD 显示最佳节点和远征胜场。
- 节点完成与远征胜利会自动更新进度。

## 数据格式

`settings.json` v1：SFX、减少动态效果。<br>
`progress.json` v1：胜场、最佳节点、已解锁道路 ID。

所有字段都会校验类型和范围。未知道路、重复解锁、负胜场和超过节点数的 best_node 都会被过滤或修正。

## 写入安全

保存流程不是直接覆盖：先完整写 `.tmp`，flush 后把旧文件改成 `.bak`，再把临时文件改成正式文件。成功才删备份；替换失败会尝试恢复旧文件。

这不能抵御存储设备物理损坏，但能降低进程中断导致半份 JSON 的概率。

## 调试记录

第一次引入 `JSON.parse_string` 时，GDScript 从 Variant 推断局部变量产生严格模式警告并按错误处理。改成显式 `Variant` 后通过。

更重要的是，Godot 在某些脚本运行错误后仍可能返回退出码 0。原测试脚本只看退出码，会出现假绿。现在同时检查 `SCRIPT ERROR`、脚本加载失败和明确的 `Godot rules: ... all passed` 标记。

## 验证

- 首次加载使用默认值。
- v0/缺版本数据迁移到 v1。
- 非法类型回到字段默认值，数字范围被夹紧，道路 ID 去重过滤。
- 设置和进度实际写入并由新服务实例读回。
- 胜场更新再次落盘。
- 损坏 JSON 实际文件回到默认进度。
- 全量测试通过后才允许提交。
