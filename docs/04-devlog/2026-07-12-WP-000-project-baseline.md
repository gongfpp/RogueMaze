# 2026-07-12：WP-000 立项与策划基线

## 目标

将“自动行进 + 拼路 + 卡牌 Roguelite”的创意整理为可执行的长期项目基线。本轮明确不创建游戏工程、不写玩法代码、不制作美术。

## 完成内容

- 定义产品愿景、目标玩家、设计支柱、阶段门禁和成功标准。
- 编写 GDD，覆盖核心循环、道路卡牌、陷阱、模式和扩展创意。
- 用 Must/Should/Could/Won't 锁定 MVP 范围。
- 建立 19–31 周参考路线图，并拆出首批细粒度工作包。
- 建立 DoR/DoD、测试层级、版本规范、风险登记和变更流程。
- 记录 Godot 4.x + GDScript 的暂定技术决策及复审条件。
- 建立开发日志、教学文档和工作包模板。

## 关键决策

- 首版采用竖屏单手操作、单机短局和几何占位美术。
- MVP 只实现工程师基础牌组、尖刺和落石，不同时开发所有创意模式。
- 玩法随机必须可用种子复现。
- 下一轮先做无引擎规则原型，避免在核心规则未明确前投资 UI 和资产。

## 验证

- 检查文档是否覆盖产品、生产、技术、日志和教学五个方面。
- 使用工作包 ID 串联路线图、Backlog、日志与下一步。
- 核对 GDD 的 MVP 描述与独立 MVP 范围表没有冲突。

## 已知风险

- 实时操作节奏和牌组循环方式尚未通过试玩验证。
- 暂无 Apple 构建环境信息，iOS 可行性需在 P2 前确认。
- 当前所有数值均为假设，不能视为平衡结论。
