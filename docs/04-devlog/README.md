# 开发日志

开发日志记录“实际发生的事”，不重复维护理想计划。每次完成或尝试一个工作包，都新增一篇 `YYYY-MM-DD-WP-xxx-title.md`。

每篇至少包含：

- 本轮目标与范围外事项。
- 实际变更。
- 验证方式与结果。
- 关键决策和偏差。
- 新风险、技术债或未决问题。
- 唯一推荐的下一工作包。
