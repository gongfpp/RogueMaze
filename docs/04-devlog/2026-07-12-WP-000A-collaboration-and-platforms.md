# 2026-07-12：WP-000A 协作规则与四平台范围

## 这轮要解决什么

项目发起人补充了六条长期要求：每轮必须 commit；使用指定 GitHub 仓库；增加 Windows/Linux；允许验证后持续开发；教程照顾零基础实习生；文档优先可读，不为格式而格式。

这轮只更新项目规则、平台范围和教学入口，不开始 `WP-001` 的规则代码。

## 实际变化

- 支持平台统一改为 Android、iOS、Windows、Linux。
- 移动端保留竖屏触摸，桌面端增加鼠标、键盘、窗口缩放和响应式布局要求。
- 明确 `origin` 地址和“每轮至少一次 commit、提交前验证、随后推送”的规则。
- 完成一个小工作包后可以继续下一个，不需要为了流程形式等待。
- 新增零基础教程，解释游戏程序、Git 和模块交接的基本概念。
- 给文档增加可读性要求：先讲能做成什么，再解释必要术语。

## 验证

- 全仓搜索旧的平台描述，更新当前有效的章程、范围、计划、流程、风险和架构文档。
- 历史日志保留当时的事实，不通过重写历史来假装项目一开始就是四平台。
- 检查 Markdown 内部链接。

## 工具限制

当前环境没有 GitHub CLI，因此不能使用自动 PR 流程。本轮仍使用原生 Git 完成本地提交、远端配置并尝试推送；若认证不可用，会保留本地 commit 并明确报告。
