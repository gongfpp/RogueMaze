# 2026-07-12：WP-032 构筑升级与移除

## 目标

节点奖励原来三选一都是“再加一张路”。牌组只会越来越厚，选择差别很小，也无法修剪不适合当前路线的牌。本轮把奖励变成真正的构筑决策，同时让升级在关卡里产生可验证的作用。

## 实际变更

- 新增 `CardState`。每张牌拥有独立实例 ID、道路 ID 和 1~2 级等级，不再只是一段道路名称。
- 固定循环牌组现在循环卡牌实例；出牌、补牌和跨节点重建手牌都会保留等级。
- 节点奖励固定呈现三类动作：加入一张内容池道路、强化一张现有牌、移除一张会拖慢循环的牌。
- 2 级牌显示 `+` 与 `ARMOR`，落到棋盘后有外圈，玩家不用记隐藏数值。
- 强化道路能吸收一次落石，随后降回 1 级；落石变为已解除，角色后来经过不会再次被判定命中。
- 奖励界面改为 `TUNE YOUR DECK`，明确标出 `ADD / UPGRADE / REMOVE`。

## 调试记录

第一次实现只让强化道路不被移除，却仍保留“已触发落石”。角色稍后进入格子时仍会失败，相当于画面说挡住了、规则却说没挡住。修复方式不是在 UI 特判，而是由玩法层给陷阱记录 `neutralized`，进入格子的规则也读取同一状态。

界面截图检查发现主场景没有被规则测试直接编译，一处反馈计时变量写错。随后用真实场景截图和全量脚本扫描补上这条验证链。

## 验证

- 覆盖卡牌升级上限、可读后缀和独立实例。
- 分别执行加牌、升级、移除，检查牌组数量与目标实例变化。
- 在落石格放置 2 级道路，确认道路降级但保留、陷阱被解除并发出表现事件。
- 三节点原有全流程继续通过。
- 405×720 奖励界面截图检查三种选择不重叠，牌型和文字可辨认。
