# 2026-07-12：WP-036 自动耐久回归

## 这轮要解决什么

单次通关测试能证明一条路径今天可用，却不容易发现“第 80 次重开后牌组没清干净”这类累积问题。本轮增加快速重复远征，让会跨节点变化的状态反复经历创建、奖励、胜利、失败和重置。

它不代替真人试玩。程序可以检查手牌是不是四张，却不能评价按钮是否顺手、节奏是否紧张或失败是否让人服气。

## 实际完成

- 新增 `tests/godot/soak_runner.gd`，默认重复 250 轮。
- 每轮跑完五个节点，并轮换加牌、升级、移除三种奖励；陷阱清空，以便把故障范围集中在远征状态机。
- 同一轮再制造一次断路失败，验证暂停不会推进倒计时、失败原因和事件正确、下一次 `reset()` 不继承旧状态。
- 持续检查节点、生命、手牌数、牌组下限、卡牌等级、实例 ID 唯一性和结算事件。
- Windows/Linux 增加独立运行入口；全量测试和 GitHub Actions 默认执行 250 轮。
