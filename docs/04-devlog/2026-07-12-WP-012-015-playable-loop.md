# 2026-07-12：WP-012~015 首个可玩拼路闭环

## 玩家现在能做什么

游戏启动后有 4 秒准备时间。玩家可以点击或拖动四张手牌到绿色合法格，点击 `ROTATE`、按 `R` 或右键旋转。角色随后自动沿道路前进；到达旗帜胜利，断路等待 1.75 秒仍没有新路则失败。空格或按钮暂停，结算后可以重开。

## 这轮实现

- `RunnerState`：位置、目标格、格内进度、断路宽限、抵达和失败。
- `GameSession`：棋盘、角色、四张手牌、补牌、倒计时、暂停和胜负。
- 响应式灰盒界面：网格、道路、角色、终点、卡牌和状态覆盖层。
- 触摸、鼠标和键盘输入统一调用同一组应用层方法。
- 合法格常驻绿色提示；拖拽到非法格显示红色；点击非法格显示具体原因。

## 调试与优化

- 初版手牌紧贴棋盘，下方留出大量空白，不适合单手操作。程序生成 405×720 截图后，将手牌移动到屏幕底部拇指区。
- 自动测试原先要求“补牌后卡牌 ID 必须变化”，但循环牌组允许连续两张同名直路。测试改为检查作者牌序和抽牌索引，避免错误约束玩法。
- Godot 直接运行测试时，新脚本尚未进入全局类缓存。全量测试现在先执行一次无窗口编辑器扫描，新克隆和 CI 也能稳定运行。
- 导出时曾把测试与开发文档打包，已通过排除规则去掉。

## 仍需真人回答

代码不能判断节奏是否有趣。`WP-016` 必须观察玩家是否看懂绿色格、是否来得及旋转、1.75 秒宽限是否太短，以及固定循环能否形成预判。
