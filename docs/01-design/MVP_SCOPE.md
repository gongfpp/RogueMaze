# MVP 范围与非目标

MVP 的目的不是证明内容足够多，而是证明“自动行进 + 实时拼路 + 循环牌组 + 一次构筑选择”能形成可重复的乐趣。

## 必须包含（Must）

- 竖屏 2D 网格，角色从左向右自动前进。
- 直路、上坡、下坡、转折、桥五类基础道路。
- 4 张手牌、抽牌/弃牌/洗回循环。
- 合法落点预览、旋转、放置、撤销未确认放置。
- 连接判定、断路/坠落失败、抵达终点胜利。
- 尖刺与落石两类陷阱。
- 3 个连续节点和一次三选一卡牌强化。
- 基础生命、暂停、重开、结算。
- 本地设置与单槽进度存档。
- Android/iOS 安全区与触屏操作；Windows/Linux 鼠标键盘和窗口缩放；四个平台完成冒烟测试。
- 基础音效、触觉开关与最小可读美术。

## 应该包含（Should）

- 专注/减速主动能力。
- 3 个预制关卡骨架及受约束变化。
- 新手引导和可跳过提示。
- 基础可访问性：色弱不只靠颜色、字号缩放预留、减少震动。
- 匿名本地调试指标；正式分析 SDK 需单独批准。

## 可以包含（Could）

- 每日固定种子挑战。
- 一次性道路和路线组合奖励。
- 第二名角色或第二套起始牌组。
- 无尽模式本地得分。

## 明确不做（Won't for MVP）

- PvP、实时联机、公会、聊天、云存档。
- UGC 关卡编辑器与关卡分享。
- 排行榜、战斗通行证、抽卡付费。
- 大量剧情、配音、复杂角色养成。
- 超过一种美术主题或超过一个完整流派。
- 动态下载内容和热更新框架。

## MVP 验收场景

1. 新安装后能在 Android/iOS 真机以及 Windows/Linux 桌面启动并进入首局。
2. 玩家能分别只用触屏或鼠标键盘完成抽牌、旋转、放置，并让角色通过高低差。
3. 玩家会因断路或生命耗尽失败，反馈能指出具体原因。
4. 玩家通过 3 个节点，完成一次卡牌强化并抵达终点。
5. 退出重进后，设置和已解锁进度仍存在。
6. 四个平台共享同一套核心规则和兼容存档格式；连续游玩 20 分钟无阻断、存档损坏或明显内存增长。

## 变更控制

新增功能必须回答：它解决哪个已观察问题？属于 Must/Should/Could 哪一层？会替换掉什么？若不能给出答案，只进入想法池，不进入当前迭代。
