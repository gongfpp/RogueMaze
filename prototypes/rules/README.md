# WP-001 无引擎规则原型

这里不是正式游戏，也不画画面。它是一套可以自动检查的最小规则，用来尽早回答：道路怎样连接、非法放置为什么失败、两种牌组循环能否稳定复现。

## 运行

需要 Node.js 20 或更高版本，不需要安装第三方依赖。

```powershell
npm test
npm run simulate:rules
```

- `npm test` 在单个进程中检查旋转、连接、越界、重叠、牌组循环和三个样例，因此在受限开发环境里也不需要额外权限。
- `npm run simulate:rules` 把三个样例的结果打印出来，便于非程序成员理解规则。

## 当前规则

- 棋盘使用从 `(0, 0)` 开始的方格坐标，`x` 向右，`y` 向下。
- 道路有左、右、上、下四种接口。
- 两个相邻方格只要一侧朝向对方开口，另一侧就必须有相反接口，否则是接口不匹配。
- 新道路至少连接一块已有道路；测试初始化起点时可以显式允许孤立放置。
- 暂时允许道路向空格保留接口，因为那代表以后可以继续施工。
- 相同牌序、操作和随机种子必须得到相同结果。

## 目录

```text
src/road.js       道路、方向和旋转
src/board.js      棋盘、放置校验和连通搜索
src/deck.js       固定循环与弃牌洗回
src/simulator.js  把牌序和放置步骤组合成一局
test/             自动测试
scenarios.js      三个可阅读的胜负样例
```

## 刻意没有做

- Godot 节点、动画、触摸或鼠标输入。
- 自动生成关卡或自动替玩家找最优解。
- 角色在道路内部的连续移动。
- 数值平衡和正式卡牌数据格式。

这些内容分别属于后续工作包。原型代码的价值是锁清规则和测试例子；进入 Godot 后会移植规则，不把 Node.js 带进游戏运行包。
