# 变更记录

本项目采用“未发布 + 里程碑版本”的记录方式。游戏尚未进入可发行版本前，所有变更都记录在 `Unreleased`。

## Unreleased

### Added

- 增加发布包 `--smoke` 模式：主场景初始化后输出固定标记并自行退出；Windows 本地构建和 Linux CI 用它验证真实二进制启动。
- 增加高低支路一次性维修台：受伤后经过回复 1 HP，并以圆环、加号、状态颜色、文字和音效反馈。
- 五个节点增加短目标提示，依次说明基础铺路、尖刺绕行、蒸汽/治疗、护甲落石和终局组合。
- 增加切后台/失焦自动暂停与保存、Android 返回键暂停，以及同时刷新设置和进度的生命周期处理。
- 增加 Ubuntu GitHub Actions：全量规则/场景测试和带缓存模板的 Linux x86_64 artifact 构建。
- 增加 Android arm64 APK 与 iOS arm64 Xcode 工程预设、四平台就绪检查、统一构建入口和 macOS iOS 构建脚本。
- 增加移动安全区布局和 1024×1024 原创纸片机械应用图标；生成提示词与 SHA-256 已入资产台账。
- 远征扩展为五节点，新增周期蒸汽喷口及最终组合尖刺、蒸汽、落石的压力核心。
- 蒸汽喷口提供开启/关闭形状、颜色和切换倒计时；内容可配置周期、开启时长、相位和伤害。
- 增加有独立实例 ID 和等级的道路卡；节点奖励可选择加牌、护甲升级或移除。
- 增加强化道路的落石护甲：吸收一次落石后降级，并通过文字、牌面标记、棋盘外圈和事件反馈呈现。
- 建立项目章程、GDD、MVP 范围、开发路线图与产品待办。
- 建立项目工作流、风险登记、架构草案和技术决策记录。
- 建立开发日志、实战教学目录和标准工作包模板。
- 增加面向零基础实习生的游戏开发与 Git 入门教程。
- 增加 WP-001 无引擎规则原型：道路端口、旋转、棋盘放置、连通搜索和胜负模拟。
- 增加固定顺序循环与基于随机种子的弃牌洗回策略。
- 增加 15 个自动测试和 3 个可直接运行的胜负样例。
- 增加“先验证规则，再进入引擎”的实战教学章节。
- 锁定 Godot 4.7 stable、GDScript 和 Compatibility 渲染。
- 创建可启动的 Godot 工程、响应式占位首页和规则演示入口。
- 将道路、棋盘、牌组与三个样例移植到 GDScript，增加 30 项 Godot 断言。
- 增加 Windows/Linux 导出预设、Windows/Linux 工具脚本和四平台工具链文档。
- 首次生成 Windows x86_64 与 Linux x86_64 桌面构建。
- 增加角色自动沿连接道路前进、断路等待与超时失败。
- 增加四张循环手牌、选牌、旋转、点击/拖拽放置和自动补牌。
- 增加合法落点、拖拽合法/非法预览和放置失败文字反馈。
- 增加暂停、倒计时、胜利、失败与重开，形成首个可玩单局闭环。
- 增加完整平路自动通关测试，Godot 断言增至 58 项。
- 增加程序化 405×720 UI 截图检查，并将手牌移到拇指操作区底部。
- 增加三节点远征结构和节点间三选一道路卡奖励。
- 增加尖刺伤害、生命值和定时落石摧毁道路。
- 增加节点/生命 HUD、首局三步教学和按原因区分的失败复盘。
- 垂直切片集成测试覆盖三节点、奖励、尖刺、落石与最终胜利，Godot 断言增至 113 项。
- 接入原创纸片机械蓝图背景，记录生成提示、用途和 SHA-256。
- 增加等比背景裁切、深色可读性蒙层、选牌抬升、角色步态、放路脉冲和落石呼吸动画。
- 增加运行时生成的放置、旋转、奖励、受伤、落石、胜利和失败音效，不引入外部音频授权风险。
- 增加 SFX 开关和游戏事件队列；Godot 断言增至 118 项。
- 增加 v1 设置和进度数据：SFX、减少动态效果、最佳节点、远征胜场和道路解锁。
- 增加临时文件、备份和原子替换写入；支持缺失、旧版、非法字段和损坏 JSON 安全回退。
- SFX 与动态效果设置即时持久化；HUD 显示最佳节点和胜场。
- 全量测试脚本现在识别 GDScript 编译/运行错误，避免引擎返回 0 时的假绿。

### Changed

- 修复 Linux CI 中 shell 脚本缺少可执行位导致的退出码 126，并在工作流中增加显式权限恢复。
- 道路、初始牌组、三节点陷阱与奖励池改由带版本号的 `gameplay.json` 提供；内容注册表会拒绝重复 ID、非法方向、未知引用、越界陷阱和中间节点空奖励。
- 道路规则、牌名、尖刺伤害、落石时间与节点奖励统一读取同一份内容，不再散落在目录和主循环的分支中。
- 目标平台从 Android/iOS 扩展为 Android、iOS、Windows、Linux。
- 每轮开发必须提交 Git，并使用 `gongfpp/RogueMaze` 远端进行版本管理。
- 开发节奏改为完成并验证一个切片后可连续领取下一切片，不设置无意义的人工暂停点。
